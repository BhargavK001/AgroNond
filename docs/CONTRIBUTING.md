
# Contributing to AgroNond

We welcome contributions to making Indian Agriculture digital!

## Workflow

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Create a **new branch** for your feature:
    ```bash
    git checkout -b feature/amazing-feature
    ```
4.  **Commit** your changes.
5.  **Push** to your branch.
6.  Open a **Pull Request**.

## Guidelines

- **Code Style**: We use ESLint. Please ensure your code is linted before committing.
- **Commits**: Use descriptive commit messages.
- **Testing**: Test your changes locally before pushing.

## Reporting Issues

If you find a bug, please open an issue with:
- Description of the bug
- Steps to reproduce
- Expected behavior
